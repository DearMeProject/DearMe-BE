## 📢 커밋 컨벤션
**형식**: 타입(기능명): 한 줄 요약

| 타입 | 설명 |
| --- | --- |
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `refactor` | 코드 리팩토링 (기능 변화 없음) |
| `chore` | 설정/환경 등 잡무 |
| `bug` | 버그 수정 |
</br>

## 🌳 Branch Naming 규칙
- **main**: 최종 배포용 브랜치 (프로덕션)
- **develop**: 개발 통합 브랜치 (테스트 서버)
- **feat/**: 새로운 기능 개발
- **bugfix/**: 버그 수정
- **release/**: 릴리즈 준비
- **hotfix/**: 긴급한 프로덕션 버그 수정
- **refactor/**: 기능 변화 없이 구조 개선
